# Current COE 程序分析

> 基于 `irom_disasm.txt` 反汇编，1271 条指令，分析程序执行流程与外设行为。

## 地址映射

| 地址 | 类型 | 用途 |
|------|------|------|
| `0x80000000` | IROM | 程序入口 |
| `0x80100000` | DRAM | ISA 测试通过计数器 |
| `0x80100004` | DRAM | ISA 测试失败计数器 |
| `0x80100008` | DRAM | ra 暂存（init_tests 用） |
| `0x80200000` | MMIO | 拨码开关 SW0（读，低 32 位） |
| `0x80200004` | MMIO | 拨码开关 SW1（读，高 32 位） |
| `0x80200020` | MMIO | 数码管 SEG（读/写） |

| `0x80200050` | MMIO | 计时器 CNT（读/写） |

## 函数表

| 地址 | 函数名（推测） | 功能 |
|------|---------------|------|
| `0x80000014` | `read_sw0()` | 读拨码开关低 32 位，返回 `sw[31:0]` |
| `0x80000040` | `read_sw1()` | 读拨码开关高 32 位，返回 `sw[63:32]` |

| `0x80000104` | `display_seg_hex(*)` | 读 DRAM[0x80100000]，转 hex，左移 24 位后写 SEG |
| `0x80000150` | `display_seg_point(val)` | 将 val 转 hex，取低 24 位，OR 到 SEG 现有值上 |
| `0x800001B4` | `display_led(val)` | 将 val 直接写入 LED |
| `0x800001EC` | `show_led_pass()` | 拼接 "通过" LED 位图并写入 |
| `0x80000288` | `show_led_fail()` | 拼接 "失败" LED 位图并写入 |
| `0x80000334` | `start_counter()` | 循环 10 次写 `0x80000000` (START) 到 CNT |

| `0x80000494` | `compute(n)` | 核心算力测试，大量循环+算术，返回 checksum |
| `0x80000540` | `main()` | 主函数 |
| `0x800005A8` | `init_tests()` | ISA 指令单元测试（37 项） |
| `0x80000658+` | `test_add/sub/...` | 各指令测试函数 |

## 执行时序与外设变化

### 阶段 1：初始化 + ISA 测试

```
PC = 0x80000000
├── 设置栈指针 SP = 0x80108030
├── 调用 init_tests()
│     ├── test_add()      → pass: DRAM[0x80100000]++
│     ├── test_sub()      → pass: DRAM[0x80100000]++
│     ├── test_slt()      → pass: DRAM[0x80100000]++
│     ├── test_sltu()     → pass: DRAM[0x80100000]++
│     ├── test_and()      → pass: DRAM[0x80100000]++
│     ├── ... (共 37 个测试)
│     └── 返回
│
│   此阶段外设状态：【无变化】
│   DRAM[0x80100000] = 37（全部通过时）
│   DRAM[0x80100004] = 0 （全部通过时）
```

### 阶段 2：显示测试结果

```
├── 调用 main()
│     ├── display_seg_hex()
│     │     ├── 读 DRAM[0x80100000]  → 37 (0x25)
│     │     ├── int_to_hex_display(37) → 转显示编码
│     │     ├── 左移 24 位
│     │     └── 写 SEG (0x80200020)
│     │
│     │   ✦ 数码管变化：最高位数码管显示 "25"（= 37 的十六进制）
│     │   ✦ LED：无变化
```

### 阶段 3：启动计时器

```
│     ├── start_counter()
│     │     └── 循环 10 次: SW 0x80200050 ← 0x80000000
│     │
│     │   ✦ 计时器开始运行
│     │   ✦ 数码管 / LED：无变化
```

### 阶段 4：运行计算负载

```
│     ├── compute(100000)     // 参数 n = 0x186A0 = 100000
│     │     ├── 外层循环 i = 0 to 99999
│     │     │     ├── sum += i * 13 + (i % 7) + (i / -3)
│     │     │     └── 大量 Branch + JAL
│     │     └── 返回 checksum
│     │
│     │   ✦ 外设：无变化（纯 CPU 计算）
│     │   ✦ 此阶段耗时最长（数十万条指令）
```

### 阶段 5：校验 + 报告结果

```
│     ├── if (checksum == 0xBEF16570)
│     │
│     │   ── 正确分支 ──────────────────────
│     │   ├── report_pass()
│     │   │     ├── stop_counter()
│     │   │     │     └── 10次: SW 0x80200050 ← 0xFFFFFFFF
│     │   │     │
│     │   │     │   ✦ 计时器停止
│     │   │     │
│     │   │     ├── read_counter() → cycle_count
│     │   │     │
│     │   │     ├── display_seg_point(cycle_count)
│     │   │     │     ├── 取低 24 位
│     │   │     │     ├── OR 到 SEG 现有值
│     │   │     │     └── 写 SEG
│     │   │     │
│     │   │     │   ✦ 数码管变化：低 3 位数码管显示运行周期数
│     │   │     │   ✦ 完整数码管：高位=测试通过数 | 低位=耗时
│     │   │     │
│     │   │     ├── show_led_pass()
│     │   │     │     ├── pattern = 0x20041808
│     │   │     │     │   (bit 构建: 0x1000 | 0x20 | 0x800 | 0x400
│     │   │     │     │   | 0x8 | 0x1000 | 0x200000)
│     │   │     │     └── display_led(pattern)
│     │   │     │
│     │   │     │   ✦ LED 变化：亮起 "通过" 灯型
│     │   │     │
│     │   │     └── t6 = 1
│     │   │
│     │   ── 错误分支 ──────────────────────
│     │   ├── report_fail()
│     │   │     ├── (同上：停计时、读周期、显示)
│     │   │     ├── show_led_fail()
│     │   │     │     ├── pattern = 0x20004824
│     │   │     │     └── display_led(pattern)
│     │   │     │
│     │   │     │   ✦ LED 变化：亮起 "失败" 灯型（与通过不同）
│     │   │     │
│     │   │     └── t6 = 0
```

### 阶段 6：停机

```
│     └── 返回 _start
└── j self               // 0x80000010: 死循环
```

## 最终硬件可观察状态

| 外设 | 内容 | 含义 |
|------|------|------|
| **数码管 [高位]** | ISA 测试通过数 (hex) | 37 项全过 → 显示 `25` |
| **数码管 [低位]** | compute 运行周期数 (hex) | 耗时，周期越少越好 |
| **LED** | `0x20041808`（通过）或 `0x20004824`（失败） | 一眼区分对错 |
| **t6 寄存器** | `1` 或 `0` | 仿真中可观测的 pass/fail 标志 |

## 跳转指令统计

| 类型 | 数量 | 占比 | 主要来源 |
|------|------|------|---------|
| Branch (B-type) | 95 | 7.5% | ISA 测试判断 + compute 循环 |
| JAL | 81 | 6.4% | 函数调用 |
| JALR | 97 | 7.6% | 函数返回 (ret) + 间接跳转 |
| **总跳转** | **273** | **21.5%** | |

> **注**：compute 核心循环 (`0x4B8–0x518`) 含 1 个 BLT 循环分支 + 2 个 JAL 函数调用，
> 执行 100000 次迭代，是分支预测器的主要热点。

---

## 对分支预测器测试的参考价值

1. **循环分支** (BLT @ 0x520)：99999 次 taken + 1 次 not-taken → 简单预测器即可高命中
2. **ISA 测试分支**：每个只执行 1 次 → 静态预测占优，动态预测器无训练机会
3. **Counter 启停循环** (BGE @ 0x370/0x3C8)：仅 10 次 → 太少，不足以训练
4. **缺失场景**：无嵌套循环、无交替分支、无数据依赖分支 → 对 2-bit 预测器区分度不足
