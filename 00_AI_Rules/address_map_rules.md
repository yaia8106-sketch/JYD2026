# 地址映射硬性规则

> 本文件只记录赛题补充说明文档中的硬性地址规定。

## 1. 硬性地址来源

硬性地址以如下文档为准：

```text
01_Docs/converted/第十届集创赛_竞业达_企业命题初赛及分赛区决赛要求补充说明/chinese.md
```

除非用户明确要求变更竞赛兼容策略，否则 RTL、测试程序、SoC 集成文档必须以本文地址为准。

## 2. 赛题规定地址空间

| 区域 | 地址范围 | 规定 |
|------|----------|------|
| IROM | `0x8000_0000` ~ `0x800F_FFFF` | 指令存储地址空间 |
| IROM 实际使用区 | `0x8000_0000` ~ `0x8000_3FFF` | 16KB，只读 |
| IROM 保留区 | `0x8000_4000` ~ `0x800F_FFFF` | 留空，待扩展 |
| DRAM | `0x8010_0000` ~ `0x801F_FFFF` | 数据存储地址空间 |
| DRAM 实际使用区 | `0x8010_0000` ~ `0x8013_FFFF` | 256KB，可读写 |
| DRAM 保留区 | `0x8014_0000` ~ `0x801F_FFFF` | 留空，待扩展 |
| MMIO | `0x8020_0000` ~ `0x8020_00FF` | 外设地址空间 |

## 3. 赛题规定 MMIO

| 地址范围 | 名称 | 规定 |
|----------|------|------|
| `0x8020_0000` ~ `0x8020_0007` | SW | 只读，低 32 位为 `sw[31:0]`，高 32 位为 `sw[63:32]` |
| `0x8020_0010` ~ `0x8020_0013` | KEY | 只读，低 8 位为 KEY |
| `0x8020_0020` ~ `0x8020_0023` | SEG | 读写，写入 32 位数据到七段数码管 |
| `0x8020_0040` ~ `0x8020_0043` | LED | 只写，写入 32 位数据到 LED |
| `0x8020_0050` ~ `0x8020_0053` | CNT | 读写，写 `0x8000_0000` 开始计数，写 `0xFFFF_FFFF` 结束计数 |

约束：

- 除 DRAM 外，外设只支持 4 字节对齐访问。
- DRAM 支持 4、2、1 byte 访问，对应 `sw`、`sh`、`sb` 等 store 指令。

## 4. 后续 SoC 地址规划原则

- 新增 DDR、DMA、accelerator、HDMI/frame buffer 地址前，先写地址规划，再改 RTL。
- 新增地址不得破坏第 2 节和第 3 节的硬性地址规定。
- 若继续保留本地 MMIO，`0x8020_0000` ~ `0x8020_00FF` 的赛题规定外设优先保持本地直连。
- 若要把某些控制寄存器迁移到 AXI-Lite，必须先确认是否涉及本文硬性地址。
