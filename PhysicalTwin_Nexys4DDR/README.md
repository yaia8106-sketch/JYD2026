# PhysicalTwin_Nexys4DDR

BRAM-only Nexys 4 DDR board wrapper for the JYD2025 digital twin CPU.

This port intentionally does not use the onboard DDR2/MIG controller in the
first bring-up. Program memory and data memory are inferred from FPGA BRAM:

- IROM: two 32-bit banks, 2048 words each, covering the existing 16 KiB IROM
  address window.
- DRAM: full 256 KiB logical DRAM, 65536 x 32-bit words.

The CPU RTL is referenced directly from `../02_Design/rtl`, so CPU changes
remain shared with the simulator and other board wrappers.

`constraints/board.xdc` is the active project constraint file, adapted to the
lowercase `board_top` port names. `constraints/Nexys-4-DDR-Master.xdc` keeps
the Digilent master XDC in-tree as a reference for unused board pins.

## Board Mapping

- Board: Digilent Nexys 4 DDR, `xc7a100tcsg324-1`.
- Clock: `CLK100MHZ` on package pin `E3`.
- CPU clock: 50 MHz generated by an MMCM from the board 100 MHz clock.
- Reset: `CPU_RESETN` on package pin `C12`, active low on the board.
- Switches: `SW[15:0]` mapped into `virtual_sw[15:0]`.
- Buttons: `BTNC/BTNU/BTNL/BTNR/BTND` mapped into `virtual_key[4:0]`.
- Seven-segment display: all 8 digits, common-anode active low.
- LEDs: 16 active-high LEDs.

With reset asserted, the board shows `12345678` and LEDs show `0x5a5a`.

## Build

From the workspace root:

```bash
./PhysicalTwin_Nexys4DDR/run_build.sh
```

The default COE set is `dual_issue/current`. To use another set:

```bash
./PhysicalTwin_Nexys4DDR/run_build.sh dual_issue/src1
```

Generated Vivado files go under `PhysicalTwin_Nexys4DDR/vivado/`. Generated
memory files go under `PhysicalTwin_Nexys4DDR/generated/`.

## Notes

- This port keeps the conservative 50 MHz CPU bring-up target. The Nexys 4 DDR
  part is a speed grade `-1`; 100 MHz should be treated as a later timing
  project.
- Because full 256 KiB DRAM is available in BRAM on XC7A100T, no physical-page
  packing workaround is used here.
- The onboard DDR2 memory is deliberately unused. Adding DDR2 later requires a
  MIG-based memory controller and a DCache/memory-bus adapter.
