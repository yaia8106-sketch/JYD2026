# 顺序双发射流水线增加 EX2 的前递设计记录

本文档记录当前顺序双发射处理器在 EX 与 MEM 之间增加一级 EX2 后，
前递网络的设计思路。本文档描述的是已经讨论确认的设计方向，不表示
对应 RTL 已经完成。

当前讨论的目标不是给 EX1、EX2、MEM、WB 各复制一套完整前递网络，而是：

1. ID 负责检查所有仍在流水线中的老指令，并尽量在 ID 准备好最终操作数；
2. 已经在 ID 准备好的操作数一路随指令保存，后面不再重复前递；
3. 只有 ID 当拍拿不到、但为了避免停顿仍允许指令前进的数据，才在后级补入；
4. 后级只处理已经明确列出的少数情况，不再重新进行完整的寄存器号比较。

## 1. 流水线结构

计划中的主流水线为：

```text
ID -> EX1 -> EX2 -> MEM -> WB
```

其中：

- ID：读取寄存器堆、检查老指令、选择可用前递值、判断是否能够继续；
- EX1：完成必须较早执行的计算，例如简单 ALU 和 load/store 地址计算，并把
  访存地址送到 Cache 的 tag RAM 与数据 BRAM；
- EX2：接收数据 BRAM 的同步读结果，完成 Cache 命中判断、load 数据格式化、
  依赖晚到数据的计算，以及分支方向和目标计算；
- MEM：保存已经形成的最终结果，执行有效 store 的 Cache 写入和 store
  buffer 压入，并发布从 EX2 寄存过来的分支重定向；
- WB：写回寄存器堆。

已经确认的硬边界是：load/store 地址必须在 EX1 结束前送到 BRAM 地址端口，
Cache hit 数据必须在 EX2 周期得到并在 EX2/MEM 边界锁存。其他纯计算逻辑
后续仍可根据综合时序微调，但不能破坏这两个边界。

## 2. ID 是主要的前递判断位置

ID 对每个实际使用的源操作数独立作出以下三种决定。

### 2.1 ID 已经取得正确数据

ID 从寄存器堆或某个老生产者取得最终正确值，并在指令离开 ID 时锁存。

此后 EX1、EX2 和 MEM 直接使用随指令保存的值，不需要再次前递，也不需要
再次比较源、目的寄存器号。

### 2.2 ID 暂时取不到，但允许以后补入

ID 已经确定真正的生产者，但该生产者的结果尚未形成。若这种情况属于本文档
允许的范围，指令可以继续前进，同时为对应操作数携带一个窄标记。

这个标记至少需要说明：

- 哪个操作数尚未准备好；
- 数据来自同组 slot0，还是来自上一组老指令；
- 若上一组有两个生产槽，数据来自哪个槽。

到达 EX2 后，根据标记选择正确数据。EX2 不再重新搜索所有老指令。

### 2.3 ID 暂时取不到，也不允许以后补入

指令停在 ID，直到数据能够在 ID 取得，或者直到后续设计明确增加相应支持。

因此：

```text
后级需要补入的数据
=
ID 当拍取不到
且
设计选择不让该指令停顿的数据
```

## 3. ID 判断必须遵守的正确性规则

ID 必须选择程序顺序上距离消费者最近的老生产者。

如果有多个老指令写同一个寄存器，即使更老生产者的值已经准备好，只要更年轻
生产者尚未准备好，就不能错误地使用更老的值。

`x0`、没有真正使用的源寄存器、被禁止配对的 slot1，以及无效指令不能产生
前递请求。

“数据在 ID 已经准备好”必须表示该值能够在本拍结束的时钟沿被可靠锁存。
仅仅知道某个后级将在未来产生结果，不能视为已经准备好。

## 4. 当前决定支持的两类后级补数

当前不建设通用的后级前递网络，只支持下面两类情况。

### 4.1 同组 slot0 简单 ALU 结果送给 slot1

同一发射组中，slot1 可以依赖 slot0 的简单 ALU 结果。

第一版把下面条件同时成立的生产者定义为“简单 ALU”：

- 指令会写目的寄存器；
- 执行单元是普通整数 ALU；
- 写回值就是 ALU 结果。

按当前 `cpu_defs.sv`，这包括加、减、左右移位、有符号/无符号比较、异或、
或、或非、与。乘除法、访存地址、分支结果、特权结果和单独生成的下一条
指令地址不在其中。这个定义直接使用译码后的语义分类，不在配对逻辑中重新
识别两套 ISA 的具体指令编码。

执行方式为：

1. ID 识别同组 RAW，并为 slot1 的相关操作数保存标记；
2. slot0 在 EX1 计算结果；
3. 时钟沿将 slot0 结果与 slot1 一起送入 EX2；
4. slot1 在 EX2 选择 slot0 结果，完成自己在 EX2 所需的计算或补数。

这样不会在一个周期内串联两次 ALU 计算。

当前已有的 `slot0 ALU -> slot1 store data` 能够使用相同的局部数据来源继续
实现。第一版同时放开 `slot0 简单 ALU -> slot1 分支/间接跳转`：分支方向
和目标本来就在 EX2 计算，它与普通 ALU、store data 共用同一组 EX2
操作数来源，不再增加一种前递来源。配对逻辑仍必须阻止访存地址 RAW 和
同组 load-use。

明确不支持同组 load-use。若 slot0 是 load、slot1 读取该 load 的目的
寄存器，则禁止两条指令作为同一组前进。

### 4.2 上一组的晚结果送给当前 EX2

若生产者的结果直到 EX2 才形成，紧随其后的消费者在 ID 时无法取得该值。
消费者可以携带标记继续前进，并在自己的 EX2 从上一组当前保存在 MEM 的
结果中取数。

这里的“晚结果”包括：

- 上一条 load 返回的数据；
- 因同组 RAW 而在 EX2 才完成的 ALU 结果；
- 因 load-use 而在 EX2 才完成、随后又被下一条指令使用的 ALU 结果。

当前允许在 EX2 使用晚到数据的消费者包括普通 ALU、分支/跳转判断和
store data。load/store 地址不属于这一类。

因此不应分别建设“load 专用连续前递”和“EX2 ALU 专用连续前递”。
二者应尽量共用：

```text
上一组保存在 MEM 的最终结果 -> 当前消费者的 EX2 操作数
```

这条通路用于保持 `A -> B -> C` 连续依赖能够每拍推进，而不建立
“EX2 本拍组合结果绕回 EX1”的长路径。

## 5. 当前不解决的情况

第一版明确不解决：

- 同组 load-use；
- load/store 地址依赖一个直到老指令 EX2 结束才产生的结果；
- 同组 slot1 的 load/store 地址依赖 slot0 ALU 结果；
- 尚未准备好的乘除法结果；
- 其他未逐类确认可以推迟到 EX2 使用的特殊操作。

这些情况默认由 ID 停顿或配对策略阻止。后续若决定支持，必须单独记录新的
数据产生时刻、使用时刻和所需通路，不能直接扩大成完整后级前递网络。

分支比较、JALR 目标和普通 ALU 已经确定可以在 EX2 使用晚到数据。下一次
访存的地址必须在 EX1 送入 BRAM，因此相邻的
`load -> load/store address` 默认停顿一拍，不建设
`BRAM output -> EX1 address adder -> BRAM address` 的同周期长组合路径。

## 6. MEM 阶段原则

MEM 不重新建设完整前递网络。

- load/store 地址在进入 MEM 前已经算好并锁存；
- load 的 `rd` 是随流水线保存的目的寄存器编号，不是需要前递的数据；
- load hit 数据已经在 EX2 完成 way 选择、字节选择和符号扩展，进入 MEM
  时就是最终寄存器值；
- load 返回的数据作为本组最终结果之一，供下一组的 EX2 使用；
- store data 必须在进入 MEM 前已经是正确值。

若 store data 在 ID 已经准备好，则一路保存到 MEM；若它属于当前允许的
两类晚到数据，则最迟在 EX2 补入并锁存。MEM 只消费已经准备好的地址和
store data，不重新读取 Cache。

一条 store 只有在确认仍然有效后，才能在 MEM 同时执行以下动作：

- 命中时写 Cache 数据 BRAM；
- 向 store buffer 压入写穿请求。

两项动作必须由同一份最终许可控制，不能发生其中一项已经执行、另一项却被
分支或异常取消的情况。

## 7. 逐拍时序

### 7.1 ID 已经取得全部操作数

```text
edge x:
- 指令进入 ID。

cycle x:
- ID 检查所有老生产者。
- ID 为每个源操作数选择寄存器堆值或已经可用的老生产者结果。

edge x+1:
- 最终操作数随指令进入 EX1。

cycle x+1:
- EX1 直接使用锁存值，后续各级不再为这些操作数做前递判断。
```

### 7.2 同组 ALU RAW

假设同组 slot0 产生 `x1`，slot1 使用 `x1` 并产生 `x2`。

```text
edge x:
- slot0、slot1 从 ID 进入 EX1。
- slot1 携带“操作数来自同组 slot0”的标记。

cycle x:
- slot0 在 EX1 计算 `x1`。
- slot1 不执行依赖 `x1` 的最终 ALU 计算。

edge x+1:
- slot0 的结果和 slot1 一起进入 EX2。

cycle x+1:
- slot1 根据标记选择 slot0 结果，并在 EX2 计算 `x2`。
```

### 7.3 相邻拍 load-use

假设 load 产生 `x1`，下一组普通 ALU 指令使用 `x1`。

```text
edge x:
- load 从 ID 进入 EX1。
- consumer 位于 ID。

cycle x:
- EX1 计算 load 地址。
- EX1 把地址送到 tag RAM 和两个 way 的数据 BRAM。
- ID 确认 consumer 的真正生产者是该 load，记录晚到标记。

edge x+1:
- 数据 BRAM 在该时钟沿采样地址。
- tag 读取结果和 load 控制进入 EX2。
- load 进入 EX2。
- consumer 携带标记进入 EX1。

cycle x+1:
- 数据 BRAM 输出两个 way 对应的 32 bit 字。
- EX2 并行比较两个 tag，并对两个 way 的数据并行完成 load 格式化。
- EX2 最后根据命中的 way 选择一次最终 load 数据。
- consumer 在 EX1 不使用尚未返回的 `x1`。

edge x+2:
- load 数据进入 MEM 的结果寄存器。
- consumer 进入 EX2。

cycle x+2:
- consumer 根据标记选择 MEM 中的 load 数据并完成 ALU 计算。
```

### 7.4 连续的晚结果依赖

假设 B 因同组 RAW 或 load-use 而在 EX2 才产生结果，下一组 C 又依赖 B。

```text
edge x:
- B 进入 EX2。
- C 携带“来自上一组”的标记进入 EX1。

cycle x:
- B 在 EX2 形成最终结果。
- C 在 EX1 不使用该晚结果。

edge x+1:
- B 的结果进入 MEM。
- C 进入 EX2。

cycle x+1:
- C 从 MEM 选择 B 的结果并完成计算。
```

### 7.5 EX2 分支和年轻 store

假设 slot0 是分支，slot1 是同组中更年轻的 store，且该分支需要重定向。

```text
edge x:
- slot0 分支和 slot1 store 进入 EX2。

cycle x:
- EX2 计算分支实际方向、目标和是否需要重定向。
- store 的地址查询可以已经发生，但它仍然没有产生写副作用。
- slot0 重定向使 slot1 store 的有效位、Cache 写许可和 store-buffer
  压入许可全部失效。

edge x+1:
- 分支重定向信息进入 MEM。
- 被取消的 slot1 store 不得作为有效 store 进入 MEM。

cycle x+1:
- MEM 向前端发布已经寄存的重定向。
- 被取消的 store 不产生 Cache 写入，也不压入 store buffer。
```

如果 slot0 是 store、slot1 是分支，则 store 更老。即使 slot1 分支需要
重定向，slot0 store 仍必须正常进入 MEM 并产生写副作用。

## 8. EX2 操作数选择

对允许在 EX2 补数的每个操作数，概念上只需要在以下来源中选择：

1. ID 已经准备并随指令保存的原值；
2. 同组 slot0 在 EX1 形成并锁存的结果；
3. 上一组当前保存在 MEM 的最终结果。

选择由 ID 产生的标记控制，而不是由 EX2 再做一套完整寄存器号比较。

两个源操作数必须能够独立选择。例如 slot1 的一个源可以来自同组 slot0，
另一个源可以来自上一组结果。

能够使用这组选数器的当前操作包括普通 ALU、分支/跳转判断和 store data。
load/store 地址在 EX1 消费，不接入这组选数器。

## 9. Cache BRAM 时序、miss 和停顿边界

当前 Cache 数据阵列是同步读 BRAM。增加 EX2 后采用以下固定时序：

```text
EX1 周期：计算地址，并驱动 tag RAM 与两个 way 的数据 BRAM
EX1/EX2 沿：数据 BRAM 采样地址，tag 结果和请求控制被锁存
EX2 周期：BRAM 输出数据，完成 tag 比较、way 选择和 load 格式化
EX2/MEM 沿：锁存最终 load 数据或 store 命中信息
MEM 周期：保存 load 结果，或执行有效 store 的写副作用
```

因此 EX2 和 MEM 不是两个独立的 Cache 读请求者，不需要在二者之间仲裁一次
读机会。连续的独立访存可以每拍由 EX1 送入一个新地址，形成每拍启动一次
查询的流水。

两个数据 RAM 是二路组相联 Cache 的两个 way，不是按地址分流的两个独立
bank。一次查询必须同时读取两个 way，之后由 tag 比较选择命中数据。

EX2 的组合路径应按并行候选加末级选择实现：

- 两个 tag 并行比较；
- 两个 way 的字节选择和符号扩展并行计算；
- 最后只根据命中的 way 选择一次格式化结果。

不应串成“先选择 way，再开始 load 格式化”的长路径。

性能建模可以先假设 Cache 始终命中，用于单独测量这两类通路的价值。

真实 RTL 仍只保留一份 load-miss 状态。若 EX2 判断老 load miss：

- 该 load 成为当前唯一 miss；
- 年轻指令保持或重放；
- 年轻访存已经送入 BRAM 的纯读结果可以作废；
- 年轻 load 不得启动第二次补行；
- 补行完成后，年轻访存重新查询；若访问同一条 line，重试通常直接命中。

store miss 继续采用 write-no-allocate 和 store buffer。若 store buffer
已满，store 和所有更年轻指令必须保持。

不可缓存和 MMIO 访存采用保守基线，不把它们当作连续 load-use 的性能目标：

- 地址仍在 EX1 形成，并在这里选择 Cache、片上 MMIO 或外部不可缓存通路；
- 若片上 MMIO 能在地址打拍后的下一周期返回，load 数据可在 EX2 接收；
- 若外部不可缓存读是可变延迟请求，该 load 保持在 EX2，所有更年轻指令
  一起保持，直到返回值有效后再进入 MEM；
- 无论走哪条通路，store 的外部写请求都只能在 MEM 获得最终年龄许可后
  发出；
- 不可缓存 load 的消费者默认等待到 ID 能取得最终值，不额外承诺
  “相邻拍不停顿”。

这样平台响应速度只影响等待周期，不改变前递网络的形状。软件模型的主性能
统计只覆盖可缓存命中路径，MMIO 和不可缓存访问另行统计等待周期。

若 MEM、EX2、EX1 可以独立停顿，还必须验证“上一组结果”和消费者标记在
停顿期间始终保持对应关系。不能只依赖正常无停顿时固定相差一拍的位置。

EX1 的真实时序路径是“已锁存操作数 -> 地址加法 -> BRAM 地址端口”；
EX2 的真实时序路径是“BRAM 输出 -> 并行格式化/命中选择 -> EX2/MEM
寄存器”。当前没有新增结构的综合报告，因此这里记录的是设计边界，不是已经
证明能够达到目标频率的结论。

## 10. 分支重定向与 store 正确性

分支的实际方向、实际目标和预测是否正确都在 EX2 计算。为保持当前 RTL
已经采用的寄存重定向时序，EX2 把重定向信息锁存到 MEM，再由 MEM 向前端
发布。因此相对当前流水线，错误预测恢复会增加一拍。

MEM 不重新计算分支，只发布已经寄存的结果。重定向只在 EX2 指令有效且能够
按流水规则推进时形成有效事件；若更老的 MEM 指令阻塞，EX2 分支必须保持，
不能重复发布重定向。

重定向必须是一次性事件，而不是“只要 MEM 中还留着分支就一直为 1”的电平。
EX2 只有在本组真正跨过 EX2/MEM 边界时才写入一次重定向事件；MEM 即使因
同组 store 或其他原因保持多拍，也不能重复冲刷流水线。分支预测器的训练
同样只在这次有效推进时发生一次，已被更老事件取消的分支不得训练预测器。

所有异常、分支重定向、Cache miss 和访存副作用统一按程序年龄处理：

- 更老指令的事件优先；同一发射组内 slot0 比 slot1 更老；
- 更老的 MEM 指令未完成时，年轻 EX2 指令的事件保持，不能越过它发布；
- 同组 slot0 分支需要重定向时，slot1 load/store 的写副作用和 miss
  分配都必须取消；
- 同组 slot0 load/store 因 miss 或 backpressure 不能前进时，slot1
  分支结果可以算出，但必须保持，不能先发布重定向；
- 同一条指令同时出现异常和正常分支重定向时，异常优先。

这组年龄规则是统一的提交许可，不应分别在分支、Cache 和 store buffer
中各写一套互不一致的优先级。

store 正确性按程序年龄处理：

| 同组 slot0 | 同组 slot1 | 需要重定向时的 store 处理 |
|---|---|---|
| 分支 | store | store 更年轻，禁止 Cache 写入和 store-buffer 压入 |
| store | 分支 | store 更老，必须正常执行 |
| 分支 | 非副作用指令 | 清除年轻 slot1 |
| 非副作用指令 | 分支 | slot0 正常完成，清除分支之后的指令 |

下一组中的年轻 store 尚未到达 MEM，由已寄存的 MEM 重定向清除其有效位，
因此不会产生写副作用。

普通分支重定向绝不能清空整个 store buffer。已经进入 store buffer 的条目
来自比分支更老且已经通过安全点的 store，必须继续排空。若未来允许投机
store 进入 buffer，则必须给条目增加年龄信息并选择性取消；第一版不采用
这种设计。

同一条有效 store 的 Cache 写使能和 store-buffer 压入必须来自同一份
年龄检查结果。错误路径 load 的 BRAM 读取本身可以作废，但它不得在重定向
同拍启动新的 miss 补行。

## 11. 软件模型的下一步

新的软件模型不从任意全连接网络开始，而按本文档的规则建模：

1. ID 对每个操作数选择正确值、晚到标记或停顿；
2. 同组 load-use 一律禁止；
3. 同组 ALU RAW 由 EX1 的 slot0 结果在 EX2 补入；
4. 上一组 load 或 EX2 ALU 晚结果由 MEM 在消费者 EX2 补入；
5. 分支/跳转允许在 EX2 使用晚到数据；
6. load/store 地址若在 EX1 不可用，则默认造成停顿；
7. Cache hit 模式下，EX1 地址到 EX2 数据固定为一拍；
8. 每次真正使用操作数时，都与顺序执行模型的正确值比较。

模型至少需要输出：

- ID 已经准备好全部操作数的指令数；
- 同组 ALU RAW 的动态次数；
- 相邻拍 load-use 的动态次数；
- 其中 consumer 是普通 ALU、分支/跳转、store data 或访存地址的分类；
- 上一组 EX2 ALU 晚结果被下一组继续使用的次数；
- 两类后级补数同时作用于不同操作数的次数；
- 因访存地址在 EX1 尚未准备好而新增的等待周期；
- 不属于已支持范围、因而必须停顿的次数和周期；
- 各项数据按六个完整程序分别统计后的结果。

## 12. 剩余事项

### 12.1 不阻塞第一版的可选扩展

当前已经没有会改变第一版主前递拓扑的待定项。以下功能只有准备继续扩大范围
时才需要重新讨论：

- 是否为乘除法晚结果增加连续前递；当前由 ID 等待结果准备好；
- 是否为不可缓存 load-use 优化等待周期；当前采用保守串行处理；
- 综合后若个别普通 ALU 操作无法在 EX1 达到目标频率，是否把它从“简单
  ALU”集合移出；当前先包含全部单周期整数 ALU 操作。

### 12.2 RTL 实现和验证事项

- load miss 检出同拍，年轻 EX1 访存的投机 BRAM 读如何作废并重试；
- Cache miss、MEM backpressure 和全流水停顿时的标记保持规则；
- store hit 写 Cache 与下一拍同地址 load 读取时，现有 store-to-load
  修正规则如何随新增级重新对齐；
- EX2 每个操作数选择器的实际输入数量、扇出和布线；
- EX2 中两个 way 并行格式化后末级选择的实际时序；
- 新结构相对当前 RTL 的最高频率、LUT 和功耗变化。

这一组不改变前递架构，只是在写 RTL、跑随机停顿回归和综合时必须逐项关闭。

最终采用哪一版 RTL，仍需经过软件完整程序统计、功能回归和 Vivado 综合。
